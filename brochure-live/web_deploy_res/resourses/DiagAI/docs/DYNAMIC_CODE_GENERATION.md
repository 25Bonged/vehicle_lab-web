# 🚀 Dynamic Code Generation System - Complete Implementation

## Overview

The system now supports **dynamic Python code generation** where the LLM can write and execute custom scripts on-the-fly to analyze signals, perform calculations, and create visualizations - just like ChatGPT Code Interpreter or Gemini Code Execution!

## ✅ What Was Implemented

### 1. **Code Execution Engine** (`bots/databot/code_executor.py`)
- Safe sandboxed Python execution environment
- Access to signal data functions (`get_signal_data`, `find_signal_name`, etc.)
- Full access to pandas, numpy, plotly, and scipy
- Automatic plot capture and conversion to base64 images
- DataFrame and data structure serialization

### 2. **New Tool Function** (`execute_code`)
- Added to `bots/databot/tools.py`
- Executes LLM-generated Python code
- Returns execution results, plots, data, and output

### 3. **LLM Integration**
- Added `execute_code` to FUNCTIONS list in `bots/databot/agent.py`
- Enhanced prompts with code generation examples
- Special response handling for code execution results

### 4. **Enhanced Prompts**
- Added code generation guidance to `bots/databot/prompts.py`
- Examples showing how to write Python code for complex queries
- Instructions on using signal access functions

## 🎯 How It Works

### User Query Flow:
```
User: "find max RPM at each gear"
    ↓
LLM analyzes query → Determines it needs custom logic
    ↓
LLM writes Python code:
    - Finds gear and RPM signals
    - Groups by gear
    - Calculates max RPM per gear
    - Creates plot
    ↓
Code executed in sandbox
    ↓
Results returned: plots, data, output
```

### Example Code Generated by LLM:
```python
import pandas as pd
import numpy as np
import plotly.graph_objects as go

# Find signals
gear_signal = find_signal_name("gear")
rpm_signal = find_signal_name("rpm")

# Get data
gear_data = get_signal_data(gear_signal)
rpm_data = get_signal_data(rpm_signal)

# Merge and analyze
df = pd.merge(gear_data, rpm_data, on='t', suffixes=('_gear', '_rpm'))
max_rpm_per_gear = df.groupby('value_gear')['value_rpm'].max()

# Create plot
fig = go.Figure()
fig.add_trace(go.Bar(x=max_rpm_per_gear.index, y=max_rpm_per_gear.values))
fig.update_layout(title='Max RPM per Gear')

# Print results
print("Max RPM per gear:")
for gear, rpm in max_rpm_per_gear.items():
    print(f"Gear {gear}: {rpm:.0f} RPM")
```

## 🔧 Available Functions in Code

When the LLM writes code, it has access to:

### Signal Access:
- `find_signal_name(query)` - Find signal by name/alias
- `get_signal_data(signal_name, trip_id=None, t0=None, t1=None)` - Get signal time series data
- `get_multiple_signals(signal_names, trip_id=None, t0=None, t1=None)` - Get multiple signals
- `get_signal_names(query="")` - List all signals
- `get_trip_ids()` - List all trip IDs
- `find_signal_by_alias(alias)` - Find signal using alias mapping

### Data Libraries:
- `pandas` (as `pd`) - Data manipulation
- `numpy` (as `np`) - Numerical operations
- `plotly.graph_objects` (as `go`) - Plotting
- `scipy` - Scientific computing (stats, signal processing, etc.)

### Database:
- `get_conn()` - Get DuckDB connection

## 🛡️ Safety Features

1. **Sandboxed Execution:**
   - No file system access (except reading)
   - No network access
   - No dangerous imports (os, subprocess, etc.)

2. **Code Sanitization:**
   - Blocks dangerous patterns (`eval`, `exec`, `compile`, etc.)
   - Prevents system command execution
   - Limits file operations

3. **Error Handling:**
   - Catches all exceptions
   - Returns detailed error messages
   - Includes traceback for debugging

## 📚 Technical Details

### Code Execution Flow:
1. LLM receives user query
2. LLM determines if custom code is needed
3. LLM writes Python code
4. Code is sanitized and executed
5. Results (plots, data, output) are captured
6. Response is formatted and returned

### Plot Handling:
- Plotly figures are automatically detected
- Converted to base64 PNG images
- Included in response as `plots` dict
- Also available as Plotly JSON for frontend

### Data Handling:
- DataFrames are serialized with shape, columns, head, and summary
- Simple data structures (dict, list) are preserved
- Complex objects are converted to strings
